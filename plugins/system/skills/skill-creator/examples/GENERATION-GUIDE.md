# 示例：使用 skill-creator 生成技能

本目录展示如何用 skill-creator 从需求生成技能。

## 示例 1：从需求生成技能

### 需求

> 用户想要一个"代码审查助手"技能，能帮助审查代码、提出改进建议、生成审查报告。

### 分析过程

1. **选择参考技能**：`web-video-presentation` 作为模式参考
2. **分析需求**：
   - 核心能力：代码审查、建议生成、报告输出
   - 触发场景：代码审查、PR review、代码质量检查
   - 目标用户：开发者

3. **应用设计模式**：
   - 阶段 + Checkpoint（审查前对齐、审查后报告）
   - 自检协议（确保审查质量）
   - 分层文档（详细规则放参考）

### 生成的技能结构

```
code-review-assistant/
├── SKILL.md
└── references/
    ├── REVIEW-STANDARDS.md  # 审查标准
    └── REPORT-FORMAT.md     # 报告格式
```

### 生成的关键文件

#### SKILL.md

```markdown
---
name: code-review-assistant
description: 代码审查助手，分析代码质量、提出改进建议、生成审查报告。Use when 用户提到代码审查、PR review、代码质量检查。
---

# Code Review Assistant

## 快速开始

当用户提供代码或 PR 链接时，执行代码审查流程。

## 工作流

```
Phase 1   接收输入
   1.1  识别代码来源
   1.2  确定审查范围
        ↓
[Checkpoint Scope] ← 对齐审查范围
        ↓
Phase 2   代码分析
   2.1  静态分析
   2.2  模式识别
   2.3  潜在问题标记
        ↓
Phase 3   生成报告
   3.1  整理发现
   3.2  优先级排序
   3.3  输出审查报告
```

## 自检清单

- [ ] 所有发现都有证据
- [ ] 建议可操作
- [ ] 优先级合理
```

## 示例 2：复杂技能

### 需求

> 用户想要一个"技术文档写作助手"，能把技术方案转成可读性强的文档。

### 应用的设计模式

- 阶段 + Checkpoint：方案确认 → 初稿 → 评审
- 单一真相源：文档结构定义放一处
- 按需渐进披露：不同阶段读不同指南

---

## 如何使用这些示例

1. 查看技能结构，理解组织方式
2. 参考 SKILL.md 写法
3. 根据自己需求调整
4. 用 skill-creator 生成时参照这个模式
