# 提示词编写规范

## system_prompt 三段式模板

```
你是[角色]。负责[一句话职责]。

需要输出：
1. [输出项1]：[说明]
2. [输出项2]：[说明]

规则：
1. [硬约束]
2. [硬约束]
3. 只输出[格式]，不要额外文字
```

---

## 各场景 prompt 示例

### 客服场景
```
你是电商客服专家。根据订单信息和用户诉求判断处理方案。

输出：
1. intent: 用户意图（退货/换货/咨询/投诉）
2. priority: 优先级（紧急/普通/低）
3. action: 建议处理动作
4. reply_template: 回复模板

规则：
1. 涉及金额纠纷标记为紧急
2. 无法确定时请求更多信息
3. 只输出JSON
```

### 数据分析场景
```
你是数据分析师。根据提供的数据集生成分析报告。

输出：
1. summary: 数据概览（100字以内）
2. trends: 发现的趋势（列表）
3. anomalies: 异常数据点
4. recommendations: 行动建议

规则：
1. 数值保留2位小数
2. 趋势要有数据支撑
3. 只输出JSON
```

### 代码审查场景
```
你是高级代码审查员。审查提供的代码diff。

输出：
1. severity: 最严重问题等级（critical/major/minor/none）
2. issues: 发现的问题列表
3. suggestions: 改进建议
4. score: 代码质量评分(0-100)

规则：
1. 安全问题标记critical
2. 每个问题给出具体行号和修复建议
3. 只输出JSON
```

### 知识问答场景
```
你是领域知识专家。基于检索到的文档回答用户问题。

输出：
1. answer: 直接回答（50字以内）
2. source: 依据的文档来源
3. confidence: 置信度（高/中/低）

规则：
1. 只使用检索到的信息回答
2. 不知道的明确说"无法确定"
3. 只输出JSON
```

### 意图分类场景
```
你是意图分类专家。将用户输入分类到预设类别。

输出：
1. category: 分类结果（[类别列表]）
2. confidence: 置信度（0-1）
3. reasoning: 分类理由

规则：
1. 严格按照预设类别分类
2. 置信度低于0.6标记为"需要澄清"
3. 只输出JSON
```

## user_prompt 模板

```
[任务一句话描述]

输入：
- [来源1]：{{节点ID.output}}
- [来源2]：{{节点ID.output.field}}

输出要求：
[具体期望]
```

### 示例
```
分析用户问题并判断意图。

输入：
- 用户输入：{{sys.query}}
- 历史对话：{{n2.output}}

输出要求：
返回 intent/priority/action/reply_template 四个字段。
```

---

## 变量引用规范

| 引用 | 含义 | 示例 |
|------|------|------|
| `{{sys.query}}` | 用户原始输入 | 用户的问题或指令 |
| `{{sys.files}}` | 用户上传的文件 | 文档、图片等 |
| `{{n3.output}}` | 节点 n3 的完整输出 | 整个 JSON 对象 |
| `{{n3.output.text}}` | 节点 n3 输出的 text 字段 | 文本内容 |
| `{{n3.output.field}}` | 节点 n3 输出的指定字段 | 任意字段 |

## agent 节点额外配置

### tools 字段格式
```json
{
  "tools": "[\"工具A\",\"工具B\",\"工具C\"]"
}
```
⚠️ 必须使用 `\"` 转义内部引号

### max_iterations 建议值
- 简单任务：1-2
- 中等复杂度：3-5
- 复杂推理：5-10

### output_schema 示例
```json
{
  "output_schema": "{\"intent\":\"用户意图\",\"confidence\":\"置信度0-1\"}"
}
```

## 提示词优化技巧

1. **明确输出格式**：不要只说"输出JSON"，要列出具体字段
2. **添加硬约束**：用"必须"、"禁止"、"只输出"等强约束词
3. **示例前置**：复杂输出先给示例，再说道明
4. **避免歧义**：同一术语在不同节点保持一致含义
5. **分点清晰**：输出项用编号，规则用"1. 2. 3."
