## 参数说明

`$ARGUMENTS` 是用户输入的参数字符串（不含 `--flag` 部分）。

- **无参数**：从头分析代码库，创建新的项目笔记（~150 行）
- **`--refine`**：读取已有项目笔记，在其基础上增补/修正，而非覆盖
- **`--limit=N`**：限定输出最大字数为 N（如 `--limit=300`）
- **`$ARGUMENTS`** 中的额外文本被视为用户的**关注重点**，例如：
  - `/project-note 关注测试架构和构建流程` → 笔记重点聚焦测试和构建
  - `/project-note --refine 补充部署文档部分` → 在原笔记基础上补充部署说明

## 执行步骤

### 1. 读取已有笔记（如有）

如果 `--refine` 标志存在，先用 `read_project_note` 工具读取当前项目笔记内容，作为完善基础。

### 2. 分析代码库

分析此代码库并编写项目笔记，包含以下内容：

1. **构建/lint/测试命令** — 特别是运行单个测试的方法
2. **代码风格规范** — 导入、格式化、类型、命名约定、错误处理等
3. **项目结构概览** — 主要目录和模块的职责
4. **关键依赖** — 框架、运行时、第三方服务
5. **开发工作流** — 如何调试、部署、CI/CD

如果有 Cursor 规则（在 `.cursor/rules/` 或 `.cursorrules` 中）或 Copilot 规则（在 `.github/copilot-instructions.md` 中），请确保包含它们。

如果 `$ARGUMENTS` 指定了关注重点，在笔记中用独立章节覆盖这些主题。

### 3. 写入笔记

用 `edit_project_note` 工具写入/更新项目笔记。

- **无 `--refine`**：全覆盖写入，内容约 150 行
- **有 `--refine`**：在原有内容基础上追加/修改，保持已有结构不变
- **有 `--limit=N`**：确保输出总行数不超过 N 个字符

### 4. 验证

最终用 `read_project_note` 读取写入的内容，确认格式正确、内容完整。
